Iterative Statements and Transfer Statements

Overview

Iterative statements run a block of code repeatedly based on a condition. Java provides three basic loop types:

  • for loop and enhanced for each loop
  • while loop
  • do while loop

Transfer statements change the normal control flow. They include break, continue, and return.

for Loop

Used when you know or expect how many iterations are required.

Syntax

for (expr1; expr2; expr3) {
    // body
}

Execution Order per Iteration

  1. expr1 runs once before the loop starts.
  2. expr2 is evaluated. If true, body runs; else loop ends.
  3. After body finishes, expr3 runs, then control returns to expr2.

Details and Rules

  • expr1, expr2, and expr3 are optional.
  • If expr2 is omitted, it becomes an infinite loop.
  • expr1 can contain a declaration. Scope is limited to the loop.
for (int i = 0, j = 10; i < j; i++, j--) { ... }

Not allowed:

for (int i = 0, int j = 0; ... ) { } // compile error

Unreachable Statement Rules

for (; false; ) {
   System.out.println("unreachable");
}
System.out.println("this is after loop");

Enhanced for Each Loop

for (ElementType var : collectionOrArray) {
    // use var
}

It is designed for iterating arrays and collections easily. Read-oriented and internally uses the iterator of the collection.

int[] arr = {1,2,3};
for (int x : arr) {
   x = x + 1; // arr unchanged
}

Concurrent modifications cause ConcurrentModificationException in fail-fast collections like ArrayList.

while Loop

Used when iteration count is unknown and condition is evaluated before the body.

int i = 0;
while (i < 5) {
    System.out.println(i);
    i++;
}

do while Loop

Guarantees at least one execution of the loop body.

int i = 10;
do {
   System.out.println("runs once");
} while (i < 5);

Transfer Statements

break

The break statement immediately exits the innermost loop or switch statement. It can also be used with a label to exit a labeled outer loop.

Use cases:

  • Stop iteration once a certain condition is met.
  • Exit nested loops using labels.
  • Terminate a switch case to prevent fall-through.
for (int i = 0; i < 10; i++) {
   if (i == 5) break; // exits loop
}

Labeled break example:

outer:
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (j == 2) break outer; // breaks outer loop
    }
}

continue

The continue statement skips the remaining statements in the current iteration and moves to the next iteration of the loop. Behavior depends on loop type:

  • In for loops, expr3 runs before the next iteration check.
  • In while and do while, control jumps to the condition check immediately.
for (int i = 0; i < 5; i++) {
    if (i == 3) continue;
    System.out.println(i);
}
// prints 0 1 2 4

Labeled continue example:

outer:
for (int i = 0; i < 3; i++) {
   for (int j = 0; j < 3; j++) {
       if (j == 1) continue outer; // skip to next i
       System.out.println(i + " " + j);
   }
}

return

The return statement immediately exits the current method. Optionally, it can return a value. It also ends all remaining statements in the method.

Use cases:

  • Exit early when a result is ready.
  • Stop execution if an error condition occurs.
int find(int[] a, int key) {
    for (int i = 0; i < a.length; i++) {
        if (a[i] == key) return i; // exit method
    }
    return -1; // key not found
}

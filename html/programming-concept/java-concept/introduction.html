Lexer & Parser — How Java source becomes a program

Overview

Different programming languages have different syntax, but the internal pipeline that turns plain source text into an executable program follows common stages. For Java (and many compiled languages) the front-end of the compiler performs lexical analysis and parsing, producing an Abstract Syntax Tree (AST) consumed by later phases.

Lexeme & Token

Lexeme — the smallest meaningful character sequence in source code (a concrete substring). Example: in int x = 10; the lexemes are int, x, =, 10, ;.

Token — a classified category for a lexeme. The lexer converts lexemes into tokens, for example: int → KEYWORD, x → IDENTIFIER, 10 → INTEGER_LITERAL.

Lexer (Lexical Analysis)

The lexer (tokenizer) reads raw source text, groups characters into lexemes and emits tokens. It usually removes whitespace and comments. The lexer does not check higher-level grammar — it only recognizes valid tokens.

Source
int x = 10;
Lexer output (tokens)
Token stream (flat)
TOKEN(KEYWORD, int) TOKEN(IDENTIFIER, x) TOKEN(OPERATOR, =) TOKEN(INTEGER_LITERAL, 10) TOKEN(SEPARATOR, ;)
Token stream (array)
[ KEYWORD(int), IDENTIFIER(x), OPERATOR(=), INTEGER_LITERAL(10), SEPARATOR(;) ]

Parser (Syntactic Analysis)

The parser consumes the token stream produced by the lexer and checks whether the sequence matches the language grammar (production rules). If tokens conform, the parser builds a syntax tree (often an AST). Otherwise it reports a syntax error.

Example simplified production:

<declaration> → <type> <identifier> = <literal> ;
Broken statement example
int = x 10;
Lexer tokens
[ KEYWORD(int), OPERATOR(=), IDENTIFIER(x), INTEGER_LITERAL(10), SEPARATOR(;) ]
Parser
Syntax error — tokens do not match any production for a declaration

Abstract Syntax Tree (AST)

The AST is a hierarchical representation of program structure. Parsers construct the AST which later phases (semantic analysis, optimization, code generation) use.

Quick Notes

  • Lexer = tokenizes input (handles comments, whitespace).
  • Parser = validates grammar & builds AST.
  • A valid token stream may still fail parsing; lexer can succeed while parser fails.
  • Errors: Lexer errors (invalid characters), Parser errors (syntax mismatch).

Common Tokens

KEYWORD, IDENTIFIER, OPERATOR, INTEGER_LITERAL, SEPARATOR

Interview-ready line

"A lexer breaks source into tokens; a parser checks those tokens against grammar and builds an AST."

Quick Recap

  • Lexeme: smallest meaningful string in source code.
  • Token: classified lexeme (keyword, identifier, literal, separator...).
  • Lexer: produces tokens from source text and strips whitespace/comments.
  • Parser: builds AST from token stream; reports syntax errors when grammar rules are violated.
